[LeetCode]339-Nested List Weight Sum

Posted on 2019-09-15 | In Leetcode Solution |

It is a frequent interview problem from LinkedIn. It is one problem and one follow-up question.

Problem Description

Given a nested list of integers, return the sum of all integers in the list weighted by their depth.
Each element is either an integer, or a list – whose elements may also be integers or other lists.

Input: [[1,1],2,[1,1]]

Output: 10

Explanation: Four 1’s at depth 2, one 2 at depth 1.

Solution Code

Thinking Behind Code

We can easily to think we should find the depth of the nested node. The exit of recurrence is when the node is integer, if is not, we recursively visit the NestedList to, every recurrence we are in, we make depth += 1. And we sum them together. It is a fairly easy way to think in the problem.

Follow-up Question

What if we change the way we assign weight. Like we make the top level the biggest weight, and the deepest level the smallest weight.

Input: [[1,1],2,[1,1]]

Output: 8

Explanation: Four 1’s at depth 1, one 2 at depth 2.

A Smart Solution

There is smart solution to solve this problem using BFS. It does not require all the hard work to obtain the depth. It uses a addition to replace multification. Essentially, every time we go deeper in nestedList, the upper element is added to the sum one more time. So we can tactfully solve the problem.
