[LeetCode]706-Design a HashMap

Posted on 2019-09-12 | In Leetcode Solution |

It is a shame that I never thought about how to implement Hashmap myself. Because I am lazy and I simply do HashMap = dict{}, Because I never expect any company would test that on a real interview. Well, now looks like LinkedIn does.

Problem Description

Design a HashMap without using any built-in hash table libraries. To be specific, your design should include these funcions:

  • put(key, value): Insert a (key,value) pair into HashMap. If the value already exists in the HashMap, update the value.
  • get(key): Returns the value to which the specified key is mapped, or -1 if this map contains no mapping for the key.
  • remove(key): Remove the mapping for the value key if this map contains the mapping for key.

Thinking Behind Code

I once took a course called Data Structure and Algorithm back in graduate school, and was told about something like collision. Therefore, the first thing came to my mind was that using an array to store the (key,value), and when collision happens, we just keep searching for an openning after that position. However, I found the linked chain to be more elegant and efficient to avoid use a huge array. It looks like this 

| A | B | C |...| J |
  |       |       |
  A1     C1      J1
  |
 A2

This can be implemented with a small array self.arr_map. When a collision happens, it will naturally expands cur.next = LinkedNode(key,value)
