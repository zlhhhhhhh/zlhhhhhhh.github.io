How to fix that

Now, we may wonder what if we just want things work as we expect? Then we can change the assignment statement as value[1] = value[:]. What value[1] = value[:] does is that it replicates the list and creates a new identical object, and changed the reference to that.

However this may work for 1-D array, it may cause serious error when there are nested arrays in the array. You can see from the below example, array b is changed even though it seems we only change the value of a. That’s because a[1] and b[1] both refers to the same object [1,2]. When a[1][0]=3 change the object, b[1][0] changes correspondingly.

Mutable Objects and Immutable Objects

In python world, objects are divided into two parts: mutable objects and immutable objects. mutable object contains: list(),set(),dict()…<br> **immutable object** contains:str, float, int, tuple…`

The mutability specifically indicates the value cannot be changed. when you want to change the variable, a new object will be created and referenced. When an immutable variable like str is reassigned a value, the address in memory will change. In contrast, when a mutable varibale like list is changedd, the address will remain the same.

Shallow Copy and Deep Copy

Shallow Copy only replicates reference, and Deep Copy can be treated as recursive replication. For immutable object, there is no difference between these two. Because you will create a new object when changing the value of an immutable object no matter you replicate reference or replicate value. However for a mutable object, Deep Copy will replicate the whole object, so it won’t affect the original version.
To call deepcopy, you can use a library copy to run copy.deepcopy(list).

Difference between = and append\extend\+=

For a list, = means assignment. It will create new object. Append\extend\+= will use the global object. If it cannot find a global object, it will throw an error.

We may as well take a little bit more about append(),extend(),+=. They both can add element to a list. append will add whatever it gives to it as a single element. The length of the list will increase by 1. extend() requires the argument to be an iterable object, and it breaks down the iterable object and append it to list one by one. The total length will increase by len(iterable_object). += works the same as extend(). The three method all won’t create new object. So the address in memory will remain the same.
